\# Day 1 - DevOps - Create a Non-Interactive Shell User\## 📌 Original Task> To accommodate the backup agent tool's specifications, the system admin team at \\\*\\\*xFusionCorp Industries\\\*\\\* requires the creation of a user with a non-interactive shell.>> \\\*\\\*Task:\\\*\\\* Create a user named `jim` with a non-interactive shell on \\\*\\\*App Server 3\\\*\\\*.\### Server Details Used| Field | Value ||---|---|| Server | App Server 3 || Hostname | `stapp03` || User | `banner` || Password | `BigGr33n` |\---\## 🖥️ Commands Used (In Order)```bash\\# 1. Connect to App Server 3 via SSHssh banner@stapp03\\# 2. Switch to root user to get the required privilegessudo su\\# 3. Check the correct path for the "nologin" shellwhich nologin\\# 4. Create the user "jim" with a non-interactive shelluseradd -s /usr/sbin/nologin jim\\# 5. Verify the user was created correctlycat /etc/passwd | grep jim```\---\## 🧠 Step-by-Step Explanation\### 1. `ssh banner@stapp03`بيعمل اتصال SSH بالسيرفر المطلوب (App Server 3) باستخدام اليوزر `banner`. الاتصال ده اللي هيسمحلي أنفذ الأوامر على السيرفر البعيد بدل ما أكون واقف قدامه فعليًا.\### 2. `sudo su`عملية إنشاء يوزر جديد (`useradd`) محتاجة صلاحيات \*\*root\*\*، واليوزر `banner` مش root بشكل افتراضي. الأمر ده بيرفع الصلاحيات عشان أقدر أنفذ أوامر الإدارة.\### 3. `which nologin`قبل ما أحدد الـ shell، حبيت أتأكد من المسار الصحيح للـ binary بتاع `nologin` على السيرفر ده تحديدًا، لأن المسار ممكن يختلف بين التوزيعات (`/sbin/nologin` أو `/usr/sbin/nologin`).\### 4. `useradd -s /usr/sbin/nologin jim`ده الأمر الأساسي في اللاب. بيعمل يوزر جديد اسمه `jim`، والـ flag `-s` بتحدد الـ \*\*default shell\*\* بتاعه يكون `/usr/sbin/nologin` بدل الشيل العادي (`/bin/bash` مثلاً). ده بيمنع أي حد يعمل login تفاعلي باليوزر ده، وهو المطلوب بالظبط عشان يستخدم مع backup agent tool اللي مش محتاج تفاعل بشري.\### 5. `cat /etc/passwd | grep jim`أمر تحقق (verification) للتأكد إن اليوزر اتعمل صح، وإن آخر عمود في السطر (اللي بيحدد الـ shell) فعلاً بيساوي `/usr/sbin/nologin`.\*\*Expected Output:\*\*---\## ⚠️ Issues Faced \& How I Solved Them| المشكلة | الحل ||---|---|| أول مرة أعمل SSH على السيرفر، ظهرت رسالة `The authenticity of host 'stapp03' can't be established` وطلب تأكيد الـ fingerprint | كتبت `yes` للموافقة على إضافة السيرفر إلى `known\_hosts`، وده أمر طبيعي أول مرة تتصل بأي سيرفر جديد. || ما كنتش متأكد 100% من المسار الصحيح لـ `nologin` على التوزيعة دي | استخدمت `which nologin` قبل تنفيذ `useradd` للتأكد من المسار الفعلي بدل ما أفترضه. || احتمال الالتباس بين استخدام `useradd` و `usermod` | حسمت الأمر إن `useradd` هي الصحيحة هنا لأن اليوزر مش موجود أصلاً (إنشاء جديد)، بينما `usermod -s` كانت هتستخدم لو اليوزر كان موجود بالفعل وعايز أغيّر الـ shell بتاعه فقط. |\---\## ✅ Resultتم إنشاء اليوزر
